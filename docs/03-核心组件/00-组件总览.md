# 核心组件总览

Agent 系统由多个核心组件协同工作。理解各组件的职责和交互关系是构建可靠 Agent 的基础。

## 组件架构图

```mermaid
graph TD
    subgraph Agent核心
        A[LLM引擎] --- B[记忆系统]
        A --- C[工具集]
        A --- D[状态管理]
        B --- E[ACI接口]
        C --- E
        D --- E
    end
    
    subgraph 外部环境
        E --- F[用户]
        C --- G[API/数据库]
        C --- H[搜索引擎]
        C --- I[代码执行器]
    end
```

## 组件交互流程

```mermaid
sequenceDiagram
    participant User as 用户
    participant ACI as ACI接口
    participant LLM as LLM引擎
    participant Memory as 记忆系统
    participant Tools as 工具集
    participant State as 状态管理

    User->>ACI: 输入请求
    ACI->>State: 创建会话状态
    ACI->>Memory: 检索相关历史
    Memory->>LLM: 提供上下文
    ACI->>LLM: 传递用户输入
    LLM->>LLM: 推理决策
    
    alt 需要工具
        LLM->>Tools: 函数调用
        Tools->>LLM: 返回结果
    end
    
    LLM->>Memory: 存储交互记录
    LLM->>State: 更新状态
    LLM->>ACI: 生成响应
    ACI->>User: 输出结果
```

## 设计原则

1. **LLM 是核心**：所有组件围绕 LLM 的推理能力构建
2. **工具是扩展**：Agent 的能力上限取决于工具集的质量
3. **记忆是上下文**：记忆系统决定了 Agent 的"连续性"
4. **ACI 是边界**：ACI 设计决定了用户体验和安全性
5. **状态是生命周期**：状态管理决定了系统的可靠性和可恢复性
